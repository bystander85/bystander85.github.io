三、远程仓库

3.1 建立SSH连接

由于你的本地Git仓库和GitHub仓库之间的传输是通过SSH加密的，所以，需要一点设置：

3.1.1 创建SSH Key

在用户主目录下，看看有没有.ssh目录，如果有，再看看这个目录下有没有id_rsa和id_rsa.pub这两个文件，如果已经有了，可直接跳到下一步。如果没有，打开Shell（Windows下打开Git Bash），创建SSH Key：

如果一切顺利的话，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人

3.1.2 上传公钥至GitHub

登陆GitHub，打开“Account settings”，“SSH and GPG keys”页面：
然后，点“New SSH key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容：

点“Add Key”，你就应该看到已经添加的Key：

为什么GitHub需要SSH Key呢？因为GitHub需要识别出你推送的提交确实是你推送的，而不是别人冒充的，而Git支持SSH协议，所以，GitHub只要知道了你的公钥，就可以确认只有你自己才能推送。

安装Hexo并发布到Github

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Hexo的安装

1、安装node.js

请查看 Ubuntu安装nodejs

2、安装git

由于Github是使用git来提交的，而生成的博客需要部署到Github。需要先下载Git
请查看 Git教程

3、安装hexo

建立软连接，变为全局,检验hexo是否已变为全局

初始化博客目录并部署到本地

新建一个空文件夹（这个就是你博客的本地目录），进入该文件夹,并初始化

2、部署之前预先生成静态文件

3、启动服务器

默认情况下，访问网址为： http://localhost:4000/

Hexo的主题切换

1、主题的选择

可以去官方的主题网站查找

2、主题的下载

我以next主题为例，可以在github中直接搜索找到这个主题
下载并解压后，在themes文件夹下要有你的主题文件，这一步就完成了

3、修改_config.yml文件中的theme值

修改改目录下的_config.yml文件
找到

将theme: landscape改为 theme: next

5、重新部署到本地即可看到主题的改变

到这，你的主题就算更换完成

Hexo美化界面

Hexo发布文章

Front-matter，这是hexo里的一个术语，Front-matter 是markdown文件最上方以 — 分隔的区域，用于指定个别文件的变量，举例来说：

将Hexo部署到github

1、在本地生成SSH公钥并绑定到github

请查看 Git教程

2、在github上创建仓库

新建一个名为 你的用户名.github.io 的仓库

比如说，如果你的github用户名是test，那么你就新建 test.github.io 的仓库（必须是你的用户名，其它名称无效），将来你的网站访问地址就是 http://test.github.io 是不是很方便？由此可见，每一个github账户最多只能创建一个这样可以直接使用域名访问的仓库。

找到 hexo 的配置文件 _config.yml 。在最下面有个 deploy 的配置，注意

repo: 换成你的仓库 git 地址

5、使用hexo -d向git提交部署
